//rules for building a parameter list:
//all parameters are within angle brackets i.e. "< >... </ >"
//the first parameter is 'parameter list'
//valid first level parameters are 'init' and 'pid'
//	each first level parameter may have the following fields 'name', 'mode', 'init'
//valid second level parameters are 'send' and 'receive'
//	each second level parameter may have the following fields 'type', 'baud', 'length' 
//	second level parameter type may be 'data'
//	second level baud and length must be integer values
//valid third level parameters are 'data'
//	third level parameter may have the following fields 'type', 'size', 'name', 'expression'
//	valid types are 'sum', 'boolean', 'int', 'double', 'hex', 'lookup', 'ASCII'
//	valid size is something like 'uint8','int8','char8' etc.
//	valid expression includes operators '(',')','exp()','^','log()','ln()','*','/','+','-','&','XOR','|','>>','<<'
//		and a denotion of the variable '#'
//valid fourth level parameters are 'lookup' which can have fields 'data' and 'conv'
//be careful because the parser (poorly) checks for significant digits after 
//every operation and it considers the number of decimal places on constant values too.
